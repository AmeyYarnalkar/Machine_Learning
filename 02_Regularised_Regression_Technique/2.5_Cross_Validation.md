

# Dataset Split

## Train-Test Split

## Definition

```id="ds1"
Dataset is divided into:
• Training Data
• Test Data
```

---

## Purpose

```id="ds2"
Training Data → used to train the model

Test Data → used to evaluate performance
on unseen data
```

---

## Example

```id="ds3"
Total data = 100 samples

Train = 80 samples
Test  = 20 samples
```

---

## Key Idea

```id="ds4"
Test data must never be used during training
```

---

## Train-Validation Split

## Definition

```id="ds5"
Training data is further split into:
• Train Set
• Validation Set
```

---

## Purpose

```id="ds6"
Train Set → used to train model

Validation Set → used to tune model
(hyperparameters, model selection)
```

---

## Example

```id="ds7"
Train data = 80 samples

Train       = 60
Validation  = 20
Test        = 20
```

---

## Key Idea

```id="ds8"
Validation helps prevent overfitting
before final testing
```

---

## Cross Validation

## Definition

```id="ds9"
Cross Validation = technique to evaluate model
by splitting data multiple times
and averaging results
```

---

## Purpose

```id="ds10"
• More reliable performance estimate
• Better use of limited data
```

---

# Types of Cross Validation

## Leave-One-Out Cross Validation (LOOCV)

## Definition

```id="ds11"
Each data point is used once as validation,
rest all as training
```

---

## Process

```id="ds12"
If dataset size = n

Run n experiments:
Each time → 1 validation, n-1 training
```

---

## Final Result

```id="ds13"
Average of all accuracies
```

---

## Advantages

```id="ds14"
• Uses almost all data for training
• Low bias
```

---

## Disadvantages

```id="ds15"
• Very slow (n experiments)
• High variance
• Computationally expensive
```

---

## Leave-P-Out Cross Validation

## Definition

```id="ds16"
Select p samples as validation
rest as training
```

---

## Key Idea

```id="ds17"
Generalization of LOOCV (p > 1)
```

---

## Disadvantages

```id="ds18"
• Extremely expensive (many combinations)
• Not practical for large datasets
```

---

## K-Fold Cross Validation

## Definition

```id="ds19"
Dataset is divided into k equal parts (folds)
```

---

## Process

```id="ds20"
1. Split data into k folds
2. For each fold:
   • Use 1 fold as validation
   • Use remaining k-1 as training
3. Repeat k times
4. Average results
```

---

## Example

```id="ds21"
Data = 100 samples
k = 5

Each fold = 20 samples
```

---

## Advantages

```id="ds22"
• Balanced train/validation split
• More reliable than single split
• Efficient use of data
```

---

## Stratified K-Fold Cross Validation

## Problem with K-Fold

```id="ds23"
Data distribution may be uneven
(e.g., class imbalance)
```

---

## Solution

```id="ds24"
Each fold maintains same class distribution
as original dataset
```

---

## Example

```id="ds25"
Dataset: 70% class A, 30% class B

Each fold:
70% A, 30% B
```

---

## Advantage

```id="ds26"
Better for classification problems
with imbalanced data
```

---

## Time Series Cross Validation

## Problem

```id="ds27"
Random splitting breaks time order
```

---

## Solution

```id="ds28"
Split data based on time sequence
```

---

## Process

```id="ds29"
Train → past data
Validation → future data
```

Example

```id="ds30"
Train: Jan → Mar
Test : Apr

Train: Jan → Apr
Test : May
```

---

## Key Idea

```id="ds31"
Always predict future using past
```

---

# Final Summary

```id="ds32"
Train-Test → final evaluation

Train-Validation → model tuning

Cross Validation → reliable performance estimation
```

---

# One-Line Understanding

```id="ds33"
Cross validation = multiple train-test splits
to get stable and reliable accuracy
```
