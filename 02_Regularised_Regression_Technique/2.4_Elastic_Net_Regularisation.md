
# Elastic Net Regularization


### 1. Why Elastic Net exists

* **Ridge problem:**
  Keeps all features (even useless ones)

* **Lasso problem:**
  Unstable when features are correlated
  (randomly picks one, ignores others)

👉 So we combine both.

---

### 2. Elastic Net Loss Function

$$
\text{Loss} = \frac{1}{2m} \sum_{i=1}^{m} (y_i - h(x_i))^2

+ \lambda_1 \sum |b_j|
+ \lambda_2 \sum b_j^2
$$

---

## # 3. Core Idea

👉 Combine:

* L1 (Lasso) → feature selection
* L2 (Ridge) → stability

---

### 4. What Elastic Net does

* Shrinks coefficients (like Ridge)
* Removes some features (like Lasso)
* Handles correlated features better

---

### Intuition (this is the real understanding)

Think like:

* Ridge = “Don’t go extreme”
* Lasso = “Remove useless stuff”

👉 Elastic Net =

> “Be stable AND remove junk”

---

### 5. Key Parameters

Instead of 2 lambdas, usually written as:

* **λ (alpha)** → overall strength
* **l1_ratio** → balance between L1 and L2

---

###  Behavior

| l1_ratio | Behavior    |
| -------- | ----------- |
| 0        | Pure Ridge  |
| 1        | Pure Lasso  |
| 0.5      | Mix of both |

---

### 6. When to use Elastic Net

👉 Use when:

* Many features
* Features are **correlated**
* You want **feature selection + stability**

---

### 7. Why it's powerful

* Doesn’t randomly drop important correlated features
* More reliable than Lasso in real datasets

---

### Trade-off

* More hyperparameters → more tuning
* Slightly more complex

---

## 🔹 8. Ridge vs Lasso vs Elastic Net

| Feature                     | Ridge | Lasso | Elastic Net |
| --------------------------- | ----- | ----- | ----------- |
| Shrinks coefficients        | ✅     | ✅     | ✅           |
| Sets coefficients to 0      | ❌     | ✅     | ✅           |
| Handles correlated features | ✅     | ❌     | ✅           |
| Feature selection           | ❌     | ✅     | ✅           |

---

### Final takeaway

* Ridge → control magnitude
* Lasso → remove features
* Elastic Net → **best of both worlds**
