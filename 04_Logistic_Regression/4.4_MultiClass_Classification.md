### Multiclass Classification using OvR (One-vs-Rest)

---

### What is OvR?

**One-vs-Rest (OvR)** is a technique to extend \
**Logistic Regression**
from binary → multiclass problems.

---

### Core Idea

If you have **K classes**, you don’t train one model.

👉 You train **K separate binary classifiers**

Each classifier learns:

> “Is this class *k* OR NOT?”

---

### Example

Classes: `Cat, Dog, Car`

You train:

* Model 1 → Cat vs (Dog + Car)
* Model 2 → Dog vs (Cat + Car)
* Model 3 → Car vs (Cat + Dog)

---

### Training Process

For each class `k`:

1. Convert labels:

   * Class `k` → 1
   * All other classes → 0

2. Train a binary logistic regression model

3. Repeat for all classes

---

### Prediction Process

Given a new input `x`:

1. Pass `x` to all K models
2. Each model outputs a probability

Example:

* Cat → 0.65
* Dog → 0.20
* Car → 0.15

3. Pick the class with **highest probability**

👉 Final prediction = **argmax(probabilities)**

---

### Mathematical View (Simple)

Each model computes:

$$
P(y = k \mid x)
$$

But these are:

* Independent estimates
* Not normalized across classes

---

### Important Characteristics

### 1. Independent Models

* Each classifier is trained separately
* No interaction between classes

---

### 2. Probabilities are not consistent

* Sum of probabilities ≠ 1
* Two classes can both have high scores

---

### 3. Simple and scalable

* Easy to implement
* Works well for many problems

---

### Advantages

* Simple to understand
* Works with any binary classifier
* Flexible (you can swap model type)

---

### Limitations

* No competition between classes
* Can give conflicting predictions
* Not theoretically optimal

---

### Confusion Matrix in OvR (Multiclass)

You still evaluate using a **multiclass confusion matrix**:

* Size → `K × K`
* Rows → actual
* Columns → predicted

###  Interpretation:

* Diagonal → correct predictions
* Off-diagonal → errors

---

### Final Mental Model

* OvR = **Break problem into multiple binary tasks**
* Each model = “one class vs all others”
* Final decision = **highest score wins**

---


##  How sklearn implements OvR internally


### Step 1: Detect multiclass problem

When you do:

```python
from sklearn.linear_model import LogisticRegression

model = LogisticRegression()
model.fit(X, y)
```

👉 sklearn checks:

* If `y` has more than 2 classes → multiclass problem

---

### Step 2: Default strategy = OvR

Unless you explicitly change it:

```python
LogisticRegression(multi_class='ovr')
```

👉 sklearn uses **OvR by default (for some solvers)**

---

### Step 3: Internally creates K binary problems

Let’s say classes = `[0, 1, 2]`

sklearn transforms it into:

| Original Label | Model 0 | Model 1 | Model 2 |
| -------------- | ------- | ------- | ------- |
| 0              | 1       | 0       | 0       |
| 1              | 0       | 1       | 0       |
| 2              | 0       | 0       | 1       |

👉 This is called **label binarization**

---

### Step 4: Train K classifiers

Internally it does something like:

```python
for each class k:
    create binary labels (k vs rest)
    train logistic regression model
```

👉 So you end up with:

* `K weight vectors`
* `K bias terms`

---

### Step 5: How prediction works

When you call:

```python
model.predict(X_test)
```

Internally:

### 1. Each classifier gives a score

```text
score_0 = model_0(x)
score_1 = model_1(x)
score_2 = model_2(x)
```

These are basically:

* sigmoid probabilities OR decision scores

---

### 2. sklearn stacks them

```text
[score_0, score_1, score_2]
```

---

### 3. Takes argmax

```python
predicted_class = argmax(scores)
```

👉 Highest score wins

---

### Important detail 

Even though it looks like **one model**, internally:

👉 It is **multiple binary models packed together**

---

### What about probabilities?

When you call:

```python
model.predict_proba(X)
```

👉 sklearn:

1. Gets probability from each classifier
2. Normalizes them (so they sum to 1)

But remember:

* These are derived from independent models
* Not truly “joint probabilities”
