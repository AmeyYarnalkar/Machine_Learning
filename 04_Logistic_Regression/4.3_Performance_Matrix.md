# Performance Metrics (Classification)


### Confusion Matrix (Binary Classification)

A **2 × 2 matrix** used to evaluate classification models.

###  Definitions

* **True Positive (TP)** → Actual = 1, Predicted = 1
* **True Negative (TN)** → Actual = 0, Predicted = 0
* **False Positive (FP)** → Actual = 0, Predicted = 1
* **False Negative (FN)** → Actual = 1, Predicted = 0

---

### Important Clarification 

There is **NO universal standard** for row/column order.

👉 Two common conventions:

### Convention 1 (most used in ML)

|             | Actual 0 | Actual 1 |
| ----------- | -------- | -------- |
| Predicted 0 | TN       | FN       |
| Predicted 1 | FP       | TP       |

### Convention 2 (some books)

|             | Actual 1 | Actual 0 |
| ----------- | -------- | -------- |
| Predicted 1 | TP       | FP       |
| Predicted 0 | FN       | TN       |

👉 **Bottom line:**
Always check labels. Don’t blindly assume.

---

## 🔹 Accuracy

$$
Accuracy = \frac{TP + TN}{TP + TN + FP + FN}
$$

### 💡 Meaning

* Overall correctness of model

---

### Problem with Accuracy (VERY IMPORTANT)

### Example (Imbalanced Dataset)

* Total samples = 1000
* 900 → Class 1
* 100 → Class 0

### Dumb model:

* Predicts **everything as 1**

👉 Accuracy = 900 / 1000 = **90%**

### Reality:

* Model is **useless for class 0**

👉 Accuracy **lies** when data is imbalanced.

---

## Precision & Recall (Core Metrics)


### Precision

$$
Precision = \frac{TP}{TP + FP}
$$

###  Intuition

> Out of all predicted positives, how many are actually correct?

### Focus

* **False Positives matter here**

---

### Recall

$$
Recall = \frac{TP}{TP + FN}
$$


### Intuition

> Out of all actual positives, how many did we correctly find?

### Focus

* **False Negatives matter**

---

### Precision vs Recall (When to use what)


#### Use Precision when:

* False positives are costly

### Examples:

* Spam detection (don’t mark important mail as spam)
* Fraud detection (don’t wrongly block users)

---

#### Use Recall when:

* False negatives are dangerous

### Examples:

* Disease detection (missing a patient is risky)
* Fire alarm systems

---


## F-Beta Score

General formula:

$$
F_\beta = \frac{(1 + \beta^2) \cdot (Precision \cdot Recall)}{(\beta^2 \cdot Precision) + Recall}
$$


### Special Case: F1 Score

$$
F1 = \frac{2 \cdot Precision \cdot Recall}{Precision + Recall}
$$

---

### Intuition of F-Beta

* It’s a **weighted balance** of Precision & Recall

---

### Role of β (beta)

* **β = 1** → Equal importance → F1
* **β > 1** → Recall is more important
* **β < 1** → Precision is more important

---

### Simple Understanding

* Precision = “How correct are my positive predictions?”
* Recall = “How many real positives did I catch?”
* F-score = “Balance between both”
