

# Logistic Regression + ROC Curve

## 1. Logistic Regression Output

* Logistic regression does **NOT directly give class labels**
* It gives **probabilities**

```text
P(y = 1 | x)
```

Example:

```text
0.2 → likely class 0  
0.7 → likely class 1
```

---

## 2. Threshold Concept (CORE IDEA)

We convert probability → class using threshold:

```text
if prob ≥ threshold → predict 1  
else → predict 0
```

Default:

```text
threshold = 0.5
```

---

## 3. Effect of Changing Threshold

### When threshold ↓ (0.5 → 0.1):

* More values classified as **1**
* TP ↑
* FP ↑

### Impact:

* **Recall (TPR)** → increases ✅
* **FPR** → increases ✅
* **Precision** → may decrease ❌

---

## 4. Key Metrics

### True Positive Rate (Recall)

```text
TPR = TP / (TP + FN)
```

### False Positive Rate

```text
FPR = FP / (FP + TN)
```

---

## 5. ROC Curve (Receiver Operating Characteristic)

👉 Plot of:

```text
X-axis → FPR  
Y-axis → TPR
```

---

## 6. How ROC is Built

We vary threshold:

```text
1.0 → 0.9 → 0.8 → ... → 0.0
```

At each threshold:

* Compute TPR
* Compute FPR
* Plot point

---

## 7. Extreme Points (VERY IMPORTANT)

### Threshold = 1.0

* No predictions = 1
* TPR = 0
* FPR = 0

👉 Point = (0, 0)

---

### Threshold = 0.0

* All predictions = 1
* TPR = 1
* FPR = 1

👉 Point = (1, 1)

---

## 8. Shape Intuition

### Good model:

* Curve bends **towards top-left**
* High TPR with low FPR

### Random model:

* Straight diagonal line

```text
(0,0) → (1,1)
```

---

## 9. Why `predict_proba` is REQUIRED

```python
model.predict(X)       # gives 0/1
model.predict_proba(X) # gives probabilities
```

👉 ROC needs multiple thresholds
👉 So we need probabilities

---

### Important Insight:

* `predict()` → only **1 point on ROC** ❌
* `predict_proba()` → full **curve** ✅

---

## 10. AUC (Area Under Curve)

👉 Measures total area under ROC curve

---

### Meaning (VERY IMPORTANT)

👉 AUC = probability that:

> A random positive gets higher score than a random negative

---

### Values:

| AUC  | Meaning           |
| ---- | ----------------- |
| 1.0  | Perfect model     |
| 0.5  | Random guessing   |
| <0.5 | Worse than random |

---

## 11. Practical Implementation (scikit-learn)

```python
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_curve, roc_auc_score

model = LogisticRegression()
model.fit(X, y)

y_probs = model.predict_proba(X)[:, 1]

fpr, tpr, thresholds = roc_curve(y, y_probs)

auc = roc_auc_score(y, y_probs)
```

---

## 12. Key Insight from Practice

* ROC changes **only when threshold crosses a probability**
* If no probability lies between thresholds → no change

---

I like where your head is going—but this **TPR/FPR idea is flawed**. Let’s fix it before it becomes a bad habit.

---

## To get the best threshold

Instead of ratio, we use:

## 👉 Youden’s J Statistic

```text
J = TPR - FPR
```

---

## Why this works

* Rewards high TPR ✅
* Penalizes high FPR ❌
* No division issues
* Balanced decision metric

---

## Practical implementation

Add this:

```python
import numpy as np

j_scores = tpr - fpr
best_index = np.argmax(j_scores)

best_threshold = thresholds[best_index]

print("Best threshold:", best_threshold)
```

---

## What this gives you

👉 The threshold where:

* You maximize correct detections
* While minimizing false alarms

---

## Think like an engineer 

There is **NO universal “best threshold”**

Depends on problem:

* Medical → minimize FN (high recall)
* Spam → minimize FP
* Fraud → depends on cost
