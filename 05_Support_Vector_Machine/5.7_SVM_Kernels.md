# Support Vector Machine Kernels 

## 1. Why kernels are needed

A basic Support Vector Machine draws a **straight line (or hyperplane)** to separate data.

This works well when the data is **linearly separable**.

Example:

```
Class A | Class B
```

But many real datasets look like this:

```
   A A A
 B       B
 B       B
   A A A
```

A straight line **cannot separate these classes**.

This is where **kernels** come in.

---

## 2. Idea behind kernels

Instead of separating data in the **current feature space**, kernels allow SVM to:

**map data into a higher dimensional space** where separation becomes easier.

Example idea:

```
2D data  → transform →  higher dimension
```

In that new space, the data may become **linearly separable**.

Then SVM simply finds a **linear boundary in that higher space**.

---

## 3. Kernel trick (main concept)

The amazing thing about SVM is that it **does not explicitly perform the transformation**.

Instead it uses a function called a **kernel function** that directly computes similarity between two data points.

This technique is known as the **kernel trick**.

Benefits:

* avoids expensive computations
* works with very high-dimensional spaces
* enables nonlinear decision boundaries

---

## 4. Common types of kernels

### Linear Kernel

Used when data is already close to linearly separable.

Characteristics:

* simplest kernel
* fastest
* often used for text classification

---

### Polynomial Kernel

Allows curved decision boundaries.

Captures interactions between features.

Useful when relationships between features are **nonlinear but structured**.

---

### RBF Kernel (Radial Basis Function / Gaussian Kernel)

Most popular kernel.

Characteristics:

* creates very flexible decision boundaries
* works well when relationship between classes is complex
* widely used in practical applications

---

### Sigmoid Kernel

Inspired by neural networks.

Less commonly used compared to other kernels.

---

## 5. What kernels really do

Kernels allow SVM to learn **nonlinear decision boundaries** without changing the original algorithm.

So SVM can behave like:

* a linear classifier
* a nonlinear classifier
* a high-dimensional classifier

depending on the kernel used.

---

## 6. Key intuition

Without kernels:

SVM finds a **straight boundary**.

With kernels:

SVM can find **complex curved boundaries** while still solving the same optimization problem.
