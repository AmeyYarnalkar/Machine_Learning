

# Bias

## Basic Meaning

```id="b1"
Bias = error due to wrong or overly simple assumptions
```

---

## Intuition

Bias is about:

```id="b2"
How far your model’s predictions are from the true pattern
```

---

## Understanding with Example

```id="b3"
Actual data → curved pattern
Predicted data → straight line
```

This means:

```id="b4"
Model cannot capture the real relationship
```

So for almost every input:

```id="b5"
Predicted value consistently deviates from actual value
```

That consistent deviation = **Bias**

---

## Important Correction (Very Important)

```id="b6"
Bias is NOT mainly caused by noise
```

Correct cause:

```id="b7"
Bias is caused by model being too simple
or wrong assumptions
```

---

## Sources of Bias

```id="b8"
• Using a simple model for complex data
• Missing important features
• Wrong assumptions (like assuming linear when data is non-linear)
```

---

## High Bias

### Meaning

```id="b9"
Model is too simple
```

---

### Behavior

```id="b10"
• Cannot capture real pattern
• Large error even on training data
```

---

### Interpretation

```id="b11"
High Bias → Underfitting
```

---

## Low Bias

### Meaning

```id="b12"
Model is flexible enough to learn pattern
```

---

### Behavior

```id="b13"
• Predictions close to actual values
• Captures underlying relationship well
```

---

## Common Misunderstanding (Fix This)

Wrong thinking:

```id="b14"
Bias is due to noise ❌
```

Correct understanding:

```id="b15"
Bias = model limitation
Noise = randomness in data (cannot be learned)
```

---

## Relation with Noise

```id="b16"
Noise affects variance more than bias
```

---

## Final Interpretation

```id="b17"
High Bias → model underfits → misses pattern

Low Bias → model captures pattern correctly
```

---

## One-Line Understanding

```id="b18"
Bias = how wrong your model is due to being too simple
```
