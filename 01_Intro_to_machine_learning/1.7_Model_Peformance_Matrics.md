# R² (R-Squared)

## Core Idea

R² measures:

```id="r2_1"
How much of the variation in output
is explained by the model
```

---

# Step-by-Step Intuition

## Step 1 — Baseline (No Model)

If you know nothing, best guess is:

```id="r2_2"
mean of all y values
```

Example

```id="r2_3"
Actual values: [100, 120, 130, 150, 200]

Mean = 140

Predictions (dumb model):
[140, 140, 140, 140, 140]
```

---

## Step 2 — Total Variance in Data

This measures how much data varies from mean.

```id="r2_4"
SS_total = Σ (yi - y_mean)^2
```

Meaning

```id="r2_5"
Total chaos in the data
before using any model
```

---

## Step 3 — Model Error

Now your model predicts:

```id="r2_6"
[110, 125, 135, 145, 190]
```

Error calculation

```id="r2_7"
SS_residual = Σ (yi - ŷi)^2
```

Meaning

```id="r2_8"
Error still remaining after using the model
```

---

## Step 4 — R² Formula

```id="r2_9"
R² = 1 - (SS_residual / SS_total)
```

---

# Interpretation

```id="r2_10"
If model explains nothing:
SS_residual ≈ SS_total → R² ≈ 0

If model is perfect:
SS_residual = 0 → R² = 1
```

---

# Meaning of Values

| R²  | Meaning                |
| --- | ---------------------- |
| 0   | Model useless          |
| 0.5 | Explains 50% variation |
| 0.8 | Explains 80% variation |
| 1   | Perfect model          |

---

# Simple Mental Model

```id="r2_11"
Total variation = 100%

Model explains = 70%
Remaining error = 30%

R² = 0.7
```

---

# The Problem with R²

R² has a flaw:

```id="r2_12"
Adding more features always increases R²
```

Even if features are useless.

Example

```id="r2_13"
Useful:
experience, education

Useless:
height, shoe size, favorite color
```

Still:

```id="r2_14"
R² increases slightly
```

Why?

```id="r2_15"
More features → more flexibility → lower error
```

But:

```id="r2_16"
Model is not actually better
```

---

# Adjusted R²

## Core Idea

```id="r2_17"
Adjusted R² penalizes unnecessary features
```

---

## Formula

```id="r2_18"
Adjusted R² =
1 - ((1 - R²) * (n - 1)) / (n - p - 1)
```

Where

```id="r2_19"
n = number of samples
p = number of features
```

---

# Intuition

```id="r2_20"
If you add features,
they must reduce error significantly
otherwise score will drop
```

---

# Behavior Comparison

| Situation           | R²        | Adjusted R² |
| ------------------- | --------- | ----------- |
| Add useful feature  | increases | increases   |
| Add useless feature | increases | decreases   |

---

# Easy Analogy

## R²

```id="r2_21"
"You got 70% marks"
```

---

## Adjusted R²

```id="r2_22"
"You got 70% marks using too many hints,
so actual score is lower"
```

---

## One-Line Summary

```id="r2_23"
R² → variance explained by model

Adjusted R² → variance explained
with penalty for extra features
```

---

## Important Reality

High R² does NOT mean good model.

Example

```id="r2_24"
R² = 0.95
```

Still bad if:

```id="r2_25"
• overfitting
• missing important features
• wrong assumptions
• non-linear data
```
