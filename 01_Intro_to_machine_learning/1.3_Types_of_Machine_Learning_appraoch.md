There are **two approaches**:

1. Instance-Based Learning (Memory-based) 
2. Model-Based Learning

---

#  Instance-Based Learning

![Image](https://miro.medium.com/v2/resize%3Afit%3A1200/0%2AOltO4Txr-D0lPWNL.png)


Instance-based learning **does not build a general model**.

Instead it simply **stores the training data** and when a new example comes, it **compares it to known examples**.

Think of it like this:

You don’t learn a formula.
You **remember examples** and compare.

Example dataset:

| Height | Weight | Label      |
| ------ | ------ | ---------- |
| 170    | 65     | Fit        |
| 180    | 80     | Fit        |
| 160    | 95     | Overweight |

Now a new person:

```
Height = 172
Weight = 70
```

The algorithm finds **similar data points** and predicts based on them.

This is exactly what **K-Nearest Neighbors (KNN)** does.

### How it works

Step 1: 
Store all training data

Step 2: 
When new data arrives

Step 3: 
Calculate distance to existing data points

Step 4: 
Find nearest neighbors

Step 5: 
Take majority vote (classification) or average (regression)

Distance formula commonly used:

```
Euclidean Distance
```

Characteristics:

✔ No training phase \
✔ Learning happens **during prediction**

This is why it's called **lazy learning**.

---

### Pros

• Very simple
• No complex training
• Works well with small datasets

### Cons

• Slow predictions
• Memory heavy
• Doesn't scale well to large datasets

---

### Algorithms using instance-based learning

* K-Nearest Neighbors (KNN)
* Locally Weighted Regression
* Case-Based Reasoning

---

#  Model-Based Learning


Model-based learning is the **opposite philosophy**.

Instead of memorizing data, the algorithm **learns a mathematical model** that represents patterns in the data.

Example:

House price prediction.

Training data:

| Size | Price |
| ---- | ----- |
| 1000 | 50    |
| 1500 | 70    |
| 2000 | 90    |

The algorithm might learn:

```
Price = 0.04 × Size + 10
```

Now the dataset can even be **discarded** because the **model captures the pattern**.

Prediction becomes:

```
Price = 0.04 × 1800 + 10
```

This is **model-based learning**.

---

### Steps in model-based learning

Step 1: 
Choose a model type

Example:

* Linear regression
* Decision tree
* Neural network

Step 2: 
Train the model using data

Step 3: 
Optimize parameters

Step 4: 
Use trained model for prediction

---

### Characteristics

✔ Learns a generalized representation \
✔ Fast prediction \
✔ Scales well

But training may be **computationally expensive**.

---

### Algorithms using model-based learning

Most ML algorithms are here:

* Linear Regression
* Logistic Regression
* Decision Trees
* Random Forest
* SVM
* Neural Networks
* Gradient Boosting

---

# Key Differences

| Feature          | Instance Based | Model Based      |
| ---------------- | -------------- | ---------------- |
| Learning style   | Memory based   | Pattern learning |
| Training phase   | Minimal        | Important        |
| Prediction speed | Slow           | Fast             |
| Memory usage     | High           | Low              |
| Generalization   | Weak           | Strong           |

---

# Simple Analogy

Instance-based learning:

```
You remember every math problem you solved
and compare new problems to old ones.
```

Model-based learning:

```
You understand the formula and apply it.
```
