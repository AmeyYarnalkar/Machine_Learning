# K-Means Clustering 

## 1. What is Clustering

Clustering is an **unsupervised learning technique** used to group
data points based on similarity.

The goal is:

> Points within the same cluster should be **similar**
> Points from different clusters should be **dissimilar**

In clustering:

* There are **no labels**
* The algorithm discovers structure automatically.

---

## 2. K-Means Clustering

K-Means is a clustering algorithm that partitions the dataset into **K clusters**.

Where:

* **K = number of groups (clusters)**

Each cluster is represented by a **centroid**.

A centroid represents the **mean position of all points in that cluster**.

---

## 3. Basic Idea

We divide the dataset into **K groups** such that:

* Points are assigned to the **nearest centroid**
* Centroids represent the **center of their cluster**

This process is repeated until clusters stabilize.

---

## 4. Algorithm Steps

### Step 1 — Initialize K Centroids

Choose **K initial centroids randomly** from the dataset.

Example:

```
Choose K = 2

Random centroids:
C1
C2
```

At this stage:

Clusters are not formed yet.

---

### Step 2 — Assign Points to Nearest Centroid

For every data point:

1. Calculate distance to each centroid
2. Assign point to the nearest centroid

Mathematically:

Let

```
x = data point
c_j = centroid j
```

Distance (Euclidean):

$$
d(x, c_j) = \sqrt{\sum_{i=1}^{n}(x_i - c_{j,i})^2}
$$

Cluster assignment:

$$
Cluster(x) = argmin_j ; d(x, c_j)
$$

Meaning:

Choose the centroid with **minimum distance**.

---

### 5. Cluster Formation

After assignment:

Each centroid has a **group of points around it**.


---

### 6. Update Centroids

For each cluster:

Compute the **mean of all points**.

That mean becomes the **new centroid**.

Mathematically:

If cluster (C_k) has (n) points:

$$
c_k = \frac{1}{n} \sum_{x \in C_k} x
$$

Meaning:

Centroid = **average of all cluster points**

---

### 7. Repeat Process

Repeat:

1. Assign points to nearest centroid
2. Recompute centroids

Until:

* Centroids stop changing
  or
* Maximum iterations reached

This is called **convergence**.

---

### 8. What K-Means Minimizes

K-Means minimizes:

### WCSS — Within Cluster Sum of Squares

Your notes highlight this concept clearly. 

WCSS measures **how compact clusters are**.

Mathematically:

$$
WCSS =
\sum_{k=1}^{K}
\sum_{x \in C_k}
||x - c_k||^2
$$

Where:

* $x$ = data point
* $c_k$ = centroid of cluster (k)

Interpretation:

Small WCSS → tighter clusters

---

## 9. Choosing the Value of K

This is one of the **main problems in K-Means**.

Your notes correctly mention:

> How to select K value? 

---

### 10. Elbow Method

To determine optimal **K**, we use **Elbow Method**.

Procedure:

1. Try K values from **1 to N**
2. Compute **WCSS for each K**
3. Plot graph:

```
K vs WCSS
```

Graph shape:

```
WCSS
 |
 |\
 | \
 |  \
 |   \__
 |       \__
 |
 +--------------> K
```

Observation:

* WCSS decreases as K increases
* But after some point improvement slows down

The **bend in the graph** is called the **Elbow Point**.

That K is selected.

---

## 11. Why WCSS Decreases

Because:

More clusters → points are closer to centroids.

Extreme case:

```
K = number of data points
```

Each point becomes its own cluster.

WCSS = 0.

But this is meaningless.

---

## 12. Problem with Random Initialization

Your notes correctly highlight this issue. 

If centroids are initialized randomly:

Problems may occur:

* Poor clustering
* Different results each run
* Algorithm may converge to bad solution

---

## 13. Solution — K-Means++

To solve bad initialization.

Idea:

Choose centroids **far apart from each other**.

Procedure:

1. Choose first centroid randomly
2. Choose next centroid far from previous ones
3. Repeat until K centroids selected

Benefit:

Better clustering stability.

---

## 14. Important Observations

### K-Means works well when:

* Clusters are **spherical**
* Clusters have **similar size**
* Data is **numerical**

---

### K-Means struggles when:

* Clusters are **non-spherical**
* Data contains **outliers**
* Clusters overlap heavily

---

## 15. Algorithm Summary

```
Input:
Dataset X
Number of clusters K

Step 1:
Initialize K centroids randomly

Step 2:
Assign each point to nearest centroid

Step 3:
Recompute centroid as mean of cluster points

Step 4:
Repeat Step 2 and Step 3 until convergence
```

---

## 16. Computational Complexity

Let:

```
N = number of points
K = clusters
D = dimensions
```

Time complexity per iteration:

$$
O(N \times K \times D)
$$
