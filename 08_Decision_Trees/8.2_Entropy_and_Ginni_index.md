## How Does a Decision Tree Choose the Best Feature to Split?

A decision tree selects the feature that **best separates the data into pure groups**.
To measure how good a split is, the algorithm uses **impurity measures**.

The two most common impurity measures are:

* **Entropy**
* **Gini Impurity**

These help determine **how mixed the classes are in a node**.

---

## 1. Entropy

### Purpose

Entropy measures the **amount of randomness or impurity in the dataset**.

* High entropy → data is **mixed**
* Low entropy → data is **pure**

Decision trees try to **reduce entropy after each split**.

Entropy comes from Information Theory and is used in algorithms like ID3 and C4.5.

---

### Entropy Formula

$Entropy(S) = - \sum_{i=1}^{c} p_i \log_2(p_i)$

### Meaning of Terms

* **S** → dataset at the node
* **c** → number of classes
* **pᵢ** → probability of class *i*

---

### When to Use Entropy

Entropy is used when the algorithm calculates **Information Gain**.

The decision tree selects the feature that **maximizes information gain**.

Commonly used in:

* ID3
* C4.5

---

### Key Idea

Entropy helps determine:

**How uncertain the dataset is.**

Values:

* **0** → perfectly pure
* **1** → maximum impurity (binary case)

---

## 2. Gini Impurity (Gini Index)

### Purpose

Gini impurity measures the **probability of incorrectly classifying a randomly chosen element** if we assign a label randomly according to class distribution.

Lower Gini → **more pure node**

Gini is mainly used in CART.

---

### Gini Formula

$Gini(S) = 1 - \sum_{i=1}^{c} p_i^2$

### Meaning

* **S** → dataset at the node
* **c** → number of classes
* **pᵢ** → probability of class *i*

---

### When to Use Gini

Used in algorithms like:

* CART
* Implementations in scikit-learn.

Most practical implementations **use Gini because it is computationally faster** than entropy.

---

## Entropy vs Gini (Quick Comparison)

| Feature        | Entropy         | Gini                  |
| -------------- | --------------- | --------------------- |
| Uses           | logarithm       | squared probabilities |
| Complexity     | slightly slower | faster                |
| Algorithms     | ID3, C4.5       | CART                  |
| Range (binary) | 0 → 1           | 0 → 0.5               |
