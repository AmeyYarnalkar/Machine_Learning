## What We Need to Fix to Avoid Overfitting in Decision Trees

To prevent overfitting, the main goal is to **stop the tree from becoming unnecessarily complex**.

A good decision tree should:

1. **Avoid very deep trees**
   Very deep trees memorize the training data instead of learning patterns.

2. **Avoid splits that only explain noise**
   Small irregularities in the data should not create new branches.

3. **Ensure each split is meaningful**
   A split should significantly reduce impurity, not just slightly.

4. **Maintain a balance between bias and variance**
   The model should be **complex enough to learn patterns** but **simple enough to generalize**.

To achieve this balance, we use a technique called **pruning**.

---

# Pruning in Decision Trees

## What is Pruning?

Pruning is the process of **removing unnecessary branches from a decision tree**.

Instead of allowing the tree to grow until it perfectly fits the training data, pruning **cuts back branches that do not contribute much to prediction accuracy**.

The idea is simple:

A **smaller and simpler tree usually generalizes better** to unseen data.

Pruning is widely used in algorithms such as

* C4.5
* CART.

---

## How Pruning Helps Reduce Overfitting

Pruning works by **eliminating splits that mainly capture noise or very small patterns in the training data**.

Benefits include:

* reduces model complexity
* improves generalization
* prevents memorization of training samples
* produces a simpler and more interpretable tree

Instead of many tiny rules, pruning keeps **only the most meaningful splits**.

---

# Types of Pruning

There are two main types of pruning used in decision trees.

---

## 1. Pre-Pruning (Early Stopping)

Pre-pruning stops the tree **before it grows too large**.

The algorithm applies conditions that **prevent further splitting**.

Common stopping conditions include:

* maximum tree depth reached
* minimum number of samples required to split
* minimum samples required in a leaf node
* minimum impurity decrease required for a split

Example:

```text
If node contains fewer than 10 samples
→ stop splitting
```

### Advantage

* computationally efficient
* prevents unnecessary tree growth

### Limitation

Sometimes the algorithm **stops too early**, missing useful patterns.

---

## 2. Post-Pruning (Backward Pruning)

Post-pruning allows the tree to **grow completely first**, then removes unnecessary branches afterward.

The process works like this:

1. build the full tree
2. evaluate each subtree
3. remove branches that do not improve model performance

If replacing a subtree with a **single leaf node does not reduce accuracy significantly**, the subtree is removed.

This creates a **simpler tree with better generalization**.

Post-pruning is used in algorithms like

* C4.5
* CART.

---

## Simple Comparison

| Method       | When it Happens         | Idea                     |
| ------------ | ----------------------- | ------------------------ |
| Pre-pruning  | Before tree fully grows | Stop early               |
| Post-pruning | After tree fully grows  | Cut unnecessary branches |

---

## Key Idea

Decision trees naturally try to **reduce impurity as much as possible**, which can create **very deep and complex trees**.

Pruning ensures that the model **remains simple enough to generalize well on unseen data**.
