## Information Gain

### Purpose

Information Gain is used to **decide which feature should be used to split the dataset in a decision tree**.

The goal is simple:

A good split should **reduce impurity in the data**.

Information Gain measures **how much uncertainty (entropy) decreases after splitting the dataset using a feature**.

Decision tree algorithms like

* ID3
* C4.5

use **Information Gain to choose the best feature**.

The feature with the **highest Information Gain** is selected for the split.

---

## Information Gain Formula
$$
IG(S, A) = Entropy(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} Entropy(S_v)
$$
---

## Formula Annotation (Meaning of Each Term)

**IG(S, A)** :
Information gain obtained by splitting dataset **S** using feature **A**.

**Entropy(S)** :
Entropy of the **entire dataset before the split**.

**Values(A)** : 
All possible values of feature **A**.

Example:
If feature = *Weather*, values could be:

* Sunny
* Rainy
* Cloudy

**Sᵥ** :
Subset of dataset where feature **A = v**.

Example:

* S_sunny
* S_rainy
* S_cloudy

**|Sᵥ| / |S|** :
Weight of that subset relative to the total dataset.

**Entropy(Sᵥ)** :
Entropy of that subset after the split.

---

## Intuition

Information Gain calculates:

```
Uncertainty before split 
      −
Weighted uncertainty after split
```

So it measures **how much the split improves purity**.

---

## What if Information Gain is High?

High information gain means:

* The feature **separates the classes very well**
* Entropy **decreases significantly**
* The split produces **purer subsets**

So the decision tree **prefers this feature**.

Example:

```
Before split entropy = 1.0
After split entropy = 0.2

Information Gain = 0.8
```

This is a **very good split**.

---

## What if Information Gain is Low?

Low information gain means:

* The split **does not reduce impurity much**
* Data remains **mixed**
* The feature is **not very useful**

Example:

```
Before split entropy = 1.0
After split entropy = 0.9

Information Gain = 0.1
```

This is a **poor split**.

---

## Summary

| Information Gain | Meaning                                |
| ---------------- | -------------------------------------- |
| High             | Feature creates very pure subsets      |
| Medium           | Feature provides moderate separation   |
| Low              | Feature does not separate classes well |

Decision trees **always choose the feature with the highest Information Gain** at each step.

---

### Note: Handling Numerical Features in Decision Trees

When a feature has **numerical (continuous) values**, it cannot be directly split like categorical features.

So the decision tree algorithm follows a specific process.

### Steps

1. **Sort the feature values** in ascending order.

2. **Generate possible split points** between consecutive values.

Example:

```
Feature values: 10, 15, 20, 30
Possible splits:

x ≤ 12.5
x ≤ 17.5
x ≤ 25
```

3. For **each possible split**, the algorithm:

   * divides the dataset into two subsets
   * computes impurity (Entropy or Gini)
   * calculates the **Information Gain**.

4. After evaluating **all possible splits**, the algorithm selects the split with the **highest Information Gain**.

---

### Important Note

This process can be **computationally expensive** because:

* every possible threshold must be tested
* entropy or Gini must be recomputed for each split
* this is done for **every numerical feature**

If a dataset has many features and many values, the number of split evaluations becomes very large.

---

### Practical Implementations

Algorithms like

* CART

optimize this process by using efficient sorting and evaluation techniques.

Libraries such as scikit-learn implement these optimizations internally.

---

### Key Takeaway

For numerical features:

```
Sort values
Generate candidate thresholds
Compute Information Gain for each
Choose the best split
```

But this **increases computational cost**, especially for large datasets.
