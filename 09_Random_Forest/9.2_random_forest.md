

# Random Forest

Random Forest is a **bagging-based ensemble technique**.

In Random Forest, the **base learners are always Decision Trees**.

Unlike general bagging where base learners can be different models like logistic regression, SVM, or regression models, in Random Forest **every base learner is a Decision Tree**.

So we build **many decision trees**, and their predictions are combined to produce the final result.

---

# Why Random Forest is Used

A single decision tree has a big problem:

It **overfits the training data**.

This means the model learns the training data very well but performs poorly on new unseen data.

Random Forest solves this problem by:

* Training **many decision trees**
* Giving each tree **different samples of data**
* Giving each tree **different subsets of features**

Because of this randomness, the trees become **different from each other**, and when their predictions are combined, the final result becomes **more stable and accurate**.

So the main purposes of Random Forest are:

* Reduce **overfitting**
* Improve **prediction accuracy**
* Reduce **variance**
* Make the model more **robust**

---

# How Random Forest Works

Step-by-step process:

1. Start with the **original dataset**.

2. Create multiple **bootstrap samples** from the dataset.

Bootstrap sampling means:

* Random sampling **with replacement**.

Example:

Original data:

```
D = {1,2,3,4,5}
```

Bootstrap sample may be:

```
{2,5,1,2,3}
```

Here some records may **repeat** and some may **not appear**.

---

3. Each bootstrap dataset is given to a **Decision Tree**.

So if we build **100 trees**, we create **100 bootstrap datasets**.

---

4. When building each tree:

Instead of considering **all features**, the tree considers only a **random subset of features** while splitting.

This introduces **feature randomness**.

---

5. Each decision tree is trained **independently and in parallel**.

---

6. After training, all trees produce predictions.

---

# Final Prediction

## For Classification

Each tree predicts a **class label**.

We take **majority voting**.

Example:

Tree outputs:

```
A, A, B, A, B
```

Final prediction:

```
A
```

---

## For Regression

Each tree predicts a **numeric value**.

We take the **average of all predictions**.

Example:

Predictions:

```
10, 12, 11, 13
```

Final prediction:

[
(10 + 12 + 11 + 13) / 4 = 11.5
]

---

# Important Concept: Feature Randomness

In Random Forest, each tree **does not use all features at every split**.

Instead, it randomly selects a **subset of features**.

Example:

If dataset has **10 features**, a tree might only consider **3 features** at a split.

This increases **diversity among trees**, which improves ensemble performance.

---

# Important Parameters of Random Forest

When implementing Random Forest (for example in **scikit-learn**), several parameters control the behavior of the model.

---

## 1. n_estimators

This parameter defines the **number of trees in the forest**.

Example:

```
n_estimators = 100
```

Means **100 decision trees** will be built.

More trees usually improve performance but increase **training time**.

---

## 2. max_depth

This controls the **maximum depth of each decision tree**.

If the depth is too large, the tree may **overfit**.

Example:

```
max_depth = 10
```

This means the tree can grow only **10 levels deep**.

---

## 3. min_samples_split

This defines the **minimum number of samples required to split a node**.

Example:

```
min_samples_split = 5
```

This means a node will split only if it contains **at least 5 samples**.

---

## 4. min_samples_leaf

This defines the **minimum number of samples required in a leaf node**.

Example:

```
min_samples_leaf = 2
```

This helps prevent the tree from creating **very small leaves**.

---

## 5. max_features

This controls the **number of features considered when searching for the best split**.

Common options:

```
sqrt
log2
None
```

Example:

```
max_features = "sqrt"
```

If dataset has **100 features**, the tree will consider **√100 = 10 features** at each split.

---

## 6. bootstrap

This parameter determines whether **bootstrap sampling is used**.

Example:

```
bootstrap = True
```

Means trees will be trained using **bootstrap samples**.

---

# Advantages of Random Forest

* Reduces **overfitting**
* Works well with **large datasets**
* Handles **high dimensional data**
* Handles **missing values**
* Provides **feature importance**

---

# Disadvantages of Random Forest

* Slower than single models
* Harder to interpret
* Requires more **memory**

---

# Simple Summary

Random Forest is an **ensemble technique based on bagging** where:

* Base learners are **decision trees**
* Trees are trained on **bootstrap samples**
* Each tree uses **random subsets of features**
* Trees are trained **in parallel**
* Final prediction is obtained using **majority voting (classification)** or **averaging (regression)**
