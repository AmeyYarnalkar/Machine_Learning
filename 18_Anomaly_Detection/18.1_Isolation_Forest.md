# Anomaly Detection Notes

## 1. What is Anomaly Detection

**Anomaly detection = Outlier detection**

It means detecting **data points that behave differently from the rest of the dataset**.

Example:

| Temperature | Label   |
| ----------- | ------- |
| 30°C        | normal  |
| 31°C        | normal  |
| 29°C        | normal  |
| 85°C        | anomaly |

The value **85°C** is unusual compared to others.

---

## 2. Goal of Anomaly Detection

The goal is to **separate abnormal data points from normal data**.

Dataset contains:

```
Normal data + Outliers
```

The algorithm tries to **identify and isolate the unusual points**.

---

## 3. Isolation Forest Idea

Isolation Forest works on one simple principle:

> **Anomalies are easier to isolate than normal points.**

Normal points are usually **clustered together**.

Outliers are **far away from the cluster**, so they get isolated quickly.

Example:

```
Normal points
[2,3,4,5,6,7]

Outlier
[100]
```

If we randomly split:

```
split < 50
```

Then

```
Left: 2,3,4,5,6,7
Right: 100
```

The outlier **gets isolated immediately**.

---

## 4. Isolation Tree Process

Isolation Forest builds many **random trees**.

Steps:

1. Select a **random feature**
2. Choose a **random split value**
3. Split the dataset
4. Continue splitting until the point is isolated

Each tree tries to **isolate points randomly**.
It continues to split until we reach each points as a leaf node

---

## 5. Path Length

**Path length = number of splits needed to isolate a point**

Example:

| Data Point | Splits Needed |
| ---------- | ------------- |
| 100        | 2             |
| 5          | 6             |
| 6          | 7             |

Observation:

```
Fewer splits → anomaly
More splits → normal
```

---

## 6. Average Path Length

Since there are many trees:

We compute the **average path length**.

$$
h(x) = \text{average path length for point } x
$$

This means:

```
average number of splits required to isolate a point
```

---

## 7. Anomaly Score Formula

Isolation Forest calculates anomaly score as:

$$
S(x,n) = 2^{-E(h(x))/c(n)}
$$

Where:

| Symbol  | Meaning                |
| ------- | ---------------------- |
| $S(x,n)$  | anomaly score          |
| $E(h(x))$ | average path length    |
| $c(n)$    | normalization constant |
| $n$       | number of data points  |

---

## 8. Normalization Constant

$$
c(n)
$$

This is the **expected average path length of normal data**.

It is used to **normalize the score** so results remain consistent across dataset sizes.

---

## 9. Interpreting Anomaly Score

Score ranges between **0 and 1**.

| Score          | Meaning        |
| -------------- | -------------- |
| close to **1** | strong anomaly |
| around **0.5** | uncertain      |
| close to **0** | normal point   |

Example:

| Point | Score | Result  |
| ----- | ----- | ------- |
| 100   | 0.95  | anomaly |
| 6     | 0.12  | normal  |

---

## 10. Key Intuition

Remember this:

```
If a point is isolated quickly → anomaly
If it takes many splits → normal
```
