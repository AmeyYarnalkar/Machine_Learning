
# DBSCAN Clustering 

## 1. What is DBSCAN

**DBSCAN** stands for:

```
Density Based Spatial Clustering of Applications with Noise
```

It is an **unsupervised clustering algorithm** that groups points based on **density of data points in a region**.

Unlike K-Means:

* DBSCAN **does not require specifying the number of clusters**
* It can detect **arbitrary shaped clusters**
* It can identify **outliers automatically**

---

## 2. Core Idea

Clusters are formed based on **density connectivity**.

If many points exist within a small region, that region is considered a **dense region**, which forms a cluster.

Points that do not belong to any dense region are considered **noise**.

---

## 3. Important Parameters

According to your notes (page 1), DBSCAN depends on **two parameters**. 

### 1️⃣ ε (epsilon)

Radius of the neighborhood around a point.

Graphically:

If we draw a **circle of radius ε around a point**, we count how many points lie inside that circle.

Mathematically:

$$
N_\varepsilon(p) =
{ q \in D \mid dist(p,q) \leq \varepsilon }
$$

Where:

* $p$ = point
* $q$ = neighboring point
* $dist(p,q)$ = distance between points

---

### 2️⃣ MinPts (minimum points)

Minimum number of points required inside the ε-neighborhood for a point to be considered **dense**.

```
MinPts ≥ number of neighbors required
```

---

## 4. Types of Points

Your notes mention **three types of points**. 

---

## Core Point

A point is a **core point** if:

$$
|N_\varepsilon(p)| \geq MinPts
$$

Meaning:

Inside the ε circle around point (p), there are **at least MinPts points**.

Core points represent **dense regions**.

---

##  Border Point

A point is a **border point** if:

* It lies within ε distance of a core point
* But it does **not have enough neighbors** to be a core point.

So it belongs to a cluster but **is not a dense center**.

---

## Noise / Outlier Point

A point is considered **noise** if:

* It is **not a core point**
* It is **not reachable from any core point**

These points are labeled:

```
cluster = -1
```

---


## DBSCAN Algorithm

### Step 1

Pick an **unvisited point**.

---

### Step 2

Find all neighbors within ε radius.

```
neighbors = points within epsilon
```

---

### Step 3

Check if point is a core point.

```
if neighbors ≥ MinPts
```

Create a **new cluster**.

---

### Step 4

Expand cluster by adding all **density reachable points**.

---

### Step 5

If point is not a core point and not reachable from a core point:

```
mark as noise
```

---


## Advantages 

### No need to specify number of clusters

Unlike K-Means.

---

###  Detects arbitrary shaped clusters

Examples:

```
moon shape
spiral
ring
snake shaped clusters
```

---

### Robust to outliers

Noise points are automatically identified.

---

##  Disadvantages

Your notes also highlight some limitations. 

###  Border points can belong to multiple clusters

Cluster assignment depends on **distance relationships**.

---

###  Struggles with varying densities

If clusters have different densities, choosing one ε becomes difficult.

---

###  Choosing ε can be difficult

Especially when **data scale is unknown**.

---


## When to Use DBSCAN

Use DBSCAN when:

* clusters have **arbitrary shapes**
* number of clusters **unknown**
* dataset contains **noise/outliers**

Examples:

```
geospatial clustering
anomaly detection
image segmentation
trajectory clustering
```

---

## Comparison with K-Means

| Property          | K-Means   | DBSCAN           |
| ----------------- | --------- | ---------------- |
| Clusters          | spherical | arbitrary shapes |
| Need K            | yes       | no               |
| Outlier detection | no        | yes              |
| Cluster size      | similar   | can vary         |

---

## Final Intuition

DBSCAN forms clusters based on **dense neighborhoods of points**.

A cluster grows by **connecting core points through chains of nearby neighbors**.

Points that do not belong to any dense region are labeled **noise**.
