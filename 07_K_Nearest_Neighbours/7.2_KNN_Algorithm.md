# K-Nearest Neighbors (KNN)

## Purpose

KNN is a **supervised learning algorithm** used for:

* **Classification** → predicting a category/label
* **Regression** → predicting a numeric value

It works based on the idea:

> Similar data points exist close to each other in feature space.

The algorithm predicts the output of a new point by looking at the **nearest training points**.

---

## Key Idea

For a new data point:

1. Measure its **distance from all training points**.
2. Select the **k closest points**.
3. Use their outputs to predict the result.

Distance is commonly measured using:

* Euclidean Distance
* Manhattan Distance

---

## KNN for Classification

## Steps

1. Choose the value of **k** (number of neighbors).
2. Compute distance between the **new point and all training points**.
3. Select the **k nearest neighbors**.
4. Count the **class labels** among those neighbors.
5. Assign the class with the **highest frequency**.

## Example

Suppose:

k = 3

Nearest neighbors of a new point:

| Neighbor | Class |
| -------- | ----- |
| 1        | A     |
| 2        | A     |
| 3        | B     |

Class counts:

* A → 2
* B → 1

Prediction → **Class A**

---

## KNN for Regression

## Steps

1. Choose **k**.
2. Compute distances from the new point.
3. Select **k nearest neighbors**.
4. Take the **average of their output values**.
5. Assign the average as the prediction.

## Example

k = 3

Neighbor outputs:

| Neighbor | Value |
| -------- | ----- |
| 1        | 10    |
| 2        | 12    |
| 3        | 14    |

Prediction:

$$
\frac{10 + 12 + 14}{3} = 12
$$

Predicted value = **12**

---

## Distance Computation

For two points (A) and (B):

Using **Euclidean distance**:

$$
D =
\sqrt{
\sum_{i=1}^{n} (x_i - y_i)^2
}
$$

Where:

* $n$ = number of features.

---

##  Time Complexity Problem

KNN is a **lazy learning algorithm**.

Meaning:

* It **does not learn a model during training**.
* It stores **all training data**.

During prediction it must:

1. Compute distance to every training point
2. Sort distances
3. Select k nearest

---

## Prediction Complexity

If:

* training samples = **N**
* features = **D**

Distance computation:

$$
O(N \times D)
$$

Sorting neighbors:

$$
O(N \log N)
$$

So prediction becomes expensive when:

* N is very large
* many predictions are required

---

## Practical Issue

If:

* training data = **1 million points**
* prediction points = **1 million**

Then total distance calculations:

$$
10^{12}
$$

This makes prediction **extremely slow**.

---

## Common Solutions

To reduce search time we use spatial data structures:

* KD-Tree
* Ball Tree

These structures organize points to allow **faster nearest neighbor search**.

Another approach:

* **Approximate nearest neighbors**

Used in large-scale ML systems.

---

## Key Advantages

* Very **simple algorithm**
* No training phase
* Works well with **small datasets**
* Non-parametric (no assumptions about data)

---

## Key Disadvantages

* Very **slow prediction**
* Requires storing **entire dataset**
* Sensitive to **feature scaling**
* Sensitive to **choice of k**
