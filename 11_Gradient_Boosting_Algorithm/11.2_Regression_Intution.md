# Gradient Boosting Regression

## Mathematical Representation

Let the dataset be:

$$
(X_1,y_1),(X_2,y_2),...,(X_n,y_n)
$$

The model prediction is built **additively**.

### Initial Model

First we start with the mean of all target values.

$$
F_0(x) = \arg\min_c \sum_{i=1}^{n}(y_i-c)^2
$$

For squared error loss, the solution becomes:

$$
F_0(x)=\bar{y}
$$

So the model initially predicts **mean of the target values**.

---

### Residuals

Residuals represent the **error made by the current model**.

$$
r_i = y_i - F_m(x_i)
$$

where

* $y_i$ = actual value
* $F_m(x_i)$ = current model prediction

---

### Decision Tree Training

A regression tree $h_m(x)$ is trained on:

$$
(X_i , r_i)
$$

So the tree learns **patterns in the errors**.

---

### Model Update

The prediction is updated by adding the tree output.

$$
F_{m+1}(x) = F_m(x) + \eta h_m(x)
$$

where

* $h_m(x)$ = prediction of tree (m)
* $\eta$ = learning rate

---

### Final Model

After (M) trees the model becomes:

$$
F(x)=F_0(x)+\sum_{m=1}^{M}\eta h_m(x)
$$

So the final prediction is **initial prediction + sum of corrections from all trees**.

---

# Intuition Behind Gradient Boosting

Gradient boosting works using a simple idea:

Instead of building one large model, we build **many small decision trees**, where each tree learns to **correct the mistakes of the previous model**.

---

## Step 1 — Start With a Simple Prediction

The model first predicts the **mean of all outputs**.

This prediction has **large error**, but it serves as a starting point.

---

## Step 2 — Compute Residuals

Residuals represent the **difference between actual value and predicted value**.

[
Residual = Actual - Prediction
]

These residuals show **how wrong the model currently is**.

---

## Step 3 — Train a Decision Tree on Residuals

Now a regression tree is trained using:

```
Input features → X
Target → residuals
```

So the tree learns **where the model is making mistakes**.

A regression tree reduces **Mean Squared Error (MSE)** by predicting the **average residual inside each leaf node**.

Since the mean minimizes squared error, the tree naturally reduces the **overall residual error**.

---

## Step 4 — Correct the Prediction

The tree predicts **corrections**.

These corrections are added to the previous prediction.

```
New Prediction = Old Prediction + Tree Correction
```

Now the prediction becomes **closer to the true value**.

---

## Step 5 — Repeat the Process

After updating predictions:

1. New residuals are calculated
2. Another tree is trained on those residuals
3. The model prediction is updated again

Each tree **reduces the remaining error step by step**.

---

## Why Error Keeps Decreasing

Each regression tree is built to **reduce Mean Squared Error of the residuals**.

As trees are added:

```
Residual error → smaller
Prediction → closer to actual value
```

So the model gradually improves.

---

## Why We Cannot Use Only the Last Tree

A common question is:

> If the last tree has the smallest error, why not just use that tree?

Because **each tree only learns a small correction**, not the full prediction.

Gradient boosting builds **weak learners**:

* Each tree is small and shallow
* Each tree corrects only a **small portion of the error**

So a single tree cannot represent the full relationship in the data.

The final prediction requires the **combined contribution of all trees**.

```
Final prediction =
Initial prediction
+ Tree1 correction
+ Tree2 correction
+ Tree3 correction
+ ...
```

Thus, the strength of gradient boosting comes from **combining many small corrections** rather than relying on a single tree.
