# Gradient Boosting for Binary Classification

## Basic Idea

The overall structure of **Gradient Boosting** remains the same as in regression.

The model still:

* builds **small decision trees sequentially**
* each tree learns to **correct the mistakes of the previous model**
* predictions improve **step by step**

The only difference is that now we are predicting **classes (0 or 1)** instead of continuous values.

---

## Initial Setup

We have:

```
Input features → X
Output → y
```

Where the output is **binary**:

```
y ∈ {0 , 1}
```

---

## Step 1 — Initialize the Prediction

In regression we start with **mean(y)**.

But in classification we cannot directly use the mean because the model optimizes **log loss**.

So first we compute the **probability of class 1**:

$$
p = \frac{\text{number of class 1 samples}}{\text{total samples}}
$$

Then we compute the **log-odds**:

$$
F_0 = \log\left(\frac{p}{1-p}\right)
$$

This value becomes the **initial model prediction score for every sample**.

---

## Step 2 — Convert Score to Probability

The model prediction is currently a **score**.

To convert it into probability we use the **sigmoid function**.

$$
p_i = \frac{1}{1 + e^{-F(x_i)}}
$$

This gives the **predicted probability of class 1**.

---

## Step 3 — Calculate Residuals

Residuals measure the **error in probability prediction**.

$$
r_i = y_i - p_i
$$

Where

```
y_i → actual class (0 or 1)
p_i → predicted probability
```

These residuals act as the **target values for the decision tree**.

---

## Step 4 — Train a Decision Tree on Residuals

A regression tree is trained using:

```
Input features → X
Target → residuals
```

The tree learns **patterns in the prediction errors**.

Each leaf predicts the **average residual**.

---

## Step 5 — Predict Residual Corrections

Now we pass the same input features through the tree.

The tree predicts **residual corrections**.

Let the tree prediction be:

$$
h_1(x)
$$

---

## Step 6 — Update the Model Prediction

The prediction score is updated by adding the tree output.

$$
F_1(x) = F_0(x) + \eta h_1(x)
$$

Where

```
η → learning rate
```

This updated score improves the model prediction.

---

## Step 7 — Convert Updated Score to Probability

Again we convert the score to probability using **sigmoid**.

$$
p = \frac{1}{1 + e^{-F_1(x)}}
$$

This gives the **updated probability of class 1**.

---

## Step 8 — Calculate Residuals for the Next Tree

Now when we build the **second decision tree**, residuals are calculated using the **updated prediction**.

$$
r_i = y_i - p_i^{(new)}
$$

Where

```
p_i(new) → probability after the first tree correction
```

These new residuals represent the **remaining classification error**.

---

## Step 9 — Train the Next Decision Tree

A new tree is trained on these updated residuals.

Again it predicts **corrections**.

The model is updated:

$$
F_2(x) = F_1(x) + \eta h_2(x)
$$

---

## Step 10 — Repeat the Process

This process repeats:

1. Compute residuals
2. Train tree on residuals
3. Update prediction score
4. Convert score to probability

Each tree **reduces the remaining classification error**.

---

# Final Prediction

After all trees are added:

1. Convert final score to probability

$$
p = \frac{1}{1 + e^{-F(x)}}
$$

2. Assign class

```
if p ≥ 0.5 → class 1
if p < 0.5 → class 0
```

---

# Key Intuition

Gradient boosting classification works by **gradually correcting probability errors**.

Instead of predicting the class directly, the model repeatedly asks:

```
How should we adjust the probability to make the prediction more accurate?
```

Each tree adds a **small correction**, and combining many corrections produces the final classifier.
